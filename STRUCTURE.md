# 📁 هيكل المشروع المنظم - معهد بنين المنصورة النموذجي

## نظرة عامة
تم إعادة تنظيم المشروع بشكل احترافي لتحسين الصيانة والتطوير المستقبلي.

## 🗂️ البنية الحالية

```
azhar1/
├── 📄 index.html                 # الصفحة الرئيسية
├── 📄 README.md                  # ملف التوثيق الأصلي
├── 📄 STRUCTURE.md               # هذا الملف (شرح الهيكل)
│
├── 📁 css/
│   └── style.css                 # جميع أنماط التصميم
│
├── 📁 js/
│   ├── app.js                    # السكريبت الرئيسي للتطبيق
│   ├── admin.js                  # سكريبت لوحة التحكم
│   └── supabase.js               # تكامل قاعدة البيانات Supabase
│
├── 📁 pages/
│   ├── admin.html                # لوحة التحكم
│   ├── about.html                # عن المعهد
│   ├── downloads.html            # صفحة التحميلات
│   ├── news.html                 # صفحة الأخبار
```

## ✅ التحديثات المنجزة

### 1️⃣ إنشاء المجلدات الرئيسية
- ✓ `css/` - لجميع ملفات التنسيق
- ✓ `js/` - لجميع ملفات JavaScript
- ✓ `pages/` - لجميع صفحات HTML الفرعية
- ✓ `assets/images/` - للصور والرسومات
- ✓ `assets/data/` - لملفات البيانات
- ✓ `config/` - للملفات الإعدادات

### 2️⃣ نقل الملفات
- ✓ نقل `style.css` → `css/style.css`
- ✓ نقل `app.js`, `admin.js`, `supabase.js` → `js/`
- ✓ نقل جميع ملفات HTML الفرعية → `pages/`

### 3️⃣ تحديث الروابط
- ✓ تحديث روابط CSS في جميع الملفات
- ✓ تحديث روابط JavaScript في جميع الملفات
- ✓ تحديث روابط الصفحات في `index.html`
- ✓ تحديث الروابط النسبية في ملفات `pages/`

## 🔗 أمثلة الروابط المحدثة

### في index.html (الجذر)
```html
<link rel="stylesheet" href="css/style.css" />
<script src="js/app.js"></script>
<a href="pages/news.html">الأخبار</a>
```

### في pages/*.html (الصفحات الفرعية)
```html
<link rel="stylesheet" href="../css/style.css" />
<script src="../js/app.js"></script>
<a href="../index.html">الرئيسية</a>
<a href="news.html">الأخبار</a>  <!-- روابط داخلية بين الصفحات -->
```

## 📝 ملاحظات مهمة

1. **الروابط النسبية**: جميع الروابط الآن نسبية وتعمل من أي مكان
2. **سهولة الصيانة**: يمكن إضافة ملفات جديدة بسهولة في المجلدات المناسبة
3. **التوسع المستقبلي**: الهيكل جاهز لإضافة:
   - صور وأيقونات في `assets/images/`
   - ملفات بيانات JSON في `assets/data/`
   - إعدادات التطبيق في `config/`

## 🚀 الخطوات التالية المقترحة

1. **إضافة صور**: ضع جميع الصور في `assets/images/` وحدّث الروابط
2. **تنظيم البيانات**: إذا كانت هناك بيانات ثابتة، ضعها في `assets/data/`
3. **إضافة ملف .gitignore**: لتجنب رفع ملفات غير ضرورية
4. **إنشاء ملف config**: للإعدادات المركزية للتطبيق

## 📞 الدعم

إذا واجهت أي مشاكل مع الروابط أو الملفات، تأكد من:
- ✓ أن جميع الملفات في المجلدات الصحيحة
- ✓ أن الروابط النسبية صحيحة (../ للصعود مستوى واحد)
- ✓ أن أسماء الملفات والمجلدات صحيحة (حساسة لحالة الأحرف على Linux)
